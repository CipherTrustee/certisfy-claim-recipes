If you are testing out PPBV with the demo certificates, you will need to create new identity anchor certificates. You can create each certificate by creating a document with a unique (uuid) id element (ex NATIONAL_ID) value.


The number of unique id anchor certificates will determine the maximum number of parties that can be part of an agreement.


Use the demo trust anchor certificate to issue your new certificates.


You will then use these certificates to make your ppbv claims.

In general, id anchor certificates should only be used for id anchoring, ie not for making claims. But it is ok to use the demo ones to make claims.


There are three actions you can perform with PPBV. You must create claims for each action before you can perform the action.

All claims for ppbv must be for private use.
The receiver id for all claims must be ppbv.certisfy.com
You must attach trust chain to all claims.


  1. Create an agreement

    Add a field pki-action, set value to ppbv-create-agreement.


    Optionally, add a field agreement-type, set value to an appropriate type. Agreement types are publicly defined and shared, meaning there will be a generally agreed upon list of agreement type names, this will ensure that lookups are performed against well known agreement type names to avoid evasiveness.


    PPBV agreement type list is maintained here.


    Optionally, add a field statement, this can be any additional information that needs to be associated with the claim.


    To submit an agreement, all parties must give their claim to a chosen leader, this party will include those claims in their own agreement claim as additional fields.


    For instance an agreement with three parties will be given to one of the three parties by the other two, the leader will then include those claims in their own agreement claim with field names such as party-2,party-3.

  2. Lookup agreements

    Add a field pki-action, set value to ppbv-lookup-agreement.


    A look up can optionally have a filter field of either agreement-type or agreement-id. A lookup without a filter will pull all agreements associated with the lookup claim owner.


    When agreement-type filter is included, its value must be set to one of the well known values defined in the agreement type list.


    Agreement type lookups are generally given to others so they can ensure you are not already party to a certain agreement type.


    A lookup with a filter for agreement-id is a way to prove to others that you are already party to a certain agreement.

  3. Terminate an agreement

    Add a field pki-action, set value to ppbv-terminate-agreement.

    An agreement can only be terminated once all parties to the agreement submit a termination claim.
